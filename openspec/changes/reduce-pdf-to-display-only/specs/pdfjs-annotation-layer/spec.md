## 新增需求

### 需求:系统必须以显示模式呈现全屏 PDF 阅读器
系统必须把全屏 PDF 阅读器作为显示模式提供，并禁止展示任何创建、编辑、删除或保存 PDF 批注的交互。

#### 场景:显示模式打开阅读器
- **当** 用户打开 PDF attachment 的全屏阅读态
- **那么** 系统必须显示 PDF 内容和阅读控制，并且不得展示标注工具栏、批注列表、保存批注按钮或批注状态消息

## 修改需求

### 需求:系统必须使用 PDF.js 渲染全屏 PDF 阅读态
系统必须在全屏 PDF 阅读态中使用 PDF.js 渲染 PDF 页面，并禁止依赖浏览器原生 PDF iframe 作为主阅读区。系统必须能够在不读取或保存 PDF 批注状态的情况下展示 PDF 内容。

#### 场景:打开 PDF.js 阅读态
- **当** 用户打开 PDF attachment 的全屏阅读态
- **那么** 系统必须通过 PDF.js 加载 `/api/attachments.file?id=<attachmentId>` 并渲染 PDF 页面

#### 场景:不使用 iframe 主阅读区
- **当** PDF 阅读态渲染
- **那么** 主 PDF 阅读区域禁止使用浏览器原生 PDF iframe 或 object 作为唯一阅读器

#### 场景:保留原文件打开入口
- **当** 用户点击打开原文件操作
- **那么** 系统必须仍然打开或下载原始 PDF attachment

#### 场景:PDF 加载失败
- **当** PDF.js 无法加载或解析 PDF
- **那么** 系统必须显示错误状态并保留打开原文件入口

### 需求:系统必须提供 PDF 页面导航和缩放
系统必须允许用户在 PDF.js 阅读态中查看多页 PDF、跳转页面并缩放页面。

#### 场景:渲染多页 PDF
- **当** PDF 包含多个页面
- **那么** 系统必须按页面顺序渲染并允许用户滚动查看所有页面

#### 场景:页面跳转
- **当** 用户输入或选择目标页码
- **那么** 系统必须滚动到对应 PDF 页面

#### 场景:缩放页面
- **当** 用户调整缩放级别
- **那么** 系统必须重新渲染页面并更新当前缩放结果

## 移除需求

### 需求:系统必须支持文本高亮标注
**Reason**: 当前版本只支持 PDF 显示，不支持用户可见的 PDF 批注创建与保存。
**Migration**: 移除文本选区创建高亮、相关工具按钮、保存逻辑和测试；未来如需恢复，必须通过新的变更重新定义需求。

### 需求:系统必须支持矩形区域标注
**Reason**: 当前版本只支持 PDF 显示，不支持矩形标注交互。
**Migration**: 移除 pointer drag 创建矩形、相关模式切换和保存逻辑；未来如需恢复，必须通过新的变更重新定义需求。

### 需求:系统必须按页面坐标稳定渲染标注
**Reason**: 当前版本不再提供用户可见的页内标注能力。
**Migration**: 删除当前显示路径中的标注坐标换算和重绘要求；若内部代码暂时保留，必须确保其不参与显示模式。

### 需求:系统必须提供标注颜色和模式工具栏
**Reason**: 当前版本只提供 PDF 阅读控制，不提供批注工具。
**Migration**: 从阅读器工具栏中移除颜色和模式选择入口，并更新相关测试。

### 需求:系统必须提供标注列表和跳转
**Reason**: 当前版本不再展示或管理批注集合。
**Migration**: 移除阅读态中的批注侧栏、删除动作和批注跳转逻辑；保留页面跳转等纯阅读控制。
